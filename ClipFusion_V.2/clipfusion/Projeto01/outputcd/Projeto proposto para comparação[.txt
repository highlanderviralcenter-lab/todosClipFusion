Projeto proposto
Nome
SignalCut Hybrid
Frase do projeto

Sistema local de descoberta, embalagem e validação de cortes virais com inteligência híbrida manual.

Missão

Pegar vídeos longos e transformar em poucos cortes altamente promissores, já com:

seleção editorial forte

score real

adaptação por plataforma

pacote social do post

aprendizado com desempenho futuro

Não é para “economizar tempo”.
É para aumentar a taxa de acerto dos cortes.

Tese central

A maioria dos sistemas erra porque tenta fazer uma destas duas coisas:

deixar tudo na mão da IA externa

deixar tudo na mão de heurística local

O projeto certo faz isto:

Sistema local

faz o que é mensurável e consistente.

IA externa

faz o que é interpretativo e editorial.

Sistema local de novo

valida, pontua e decide.

Então o coração do projeto é:

local → IA externa → local
O que o sistema realmente faz

Ele não é só um cortador.

Ele é um sistema que:

transcreve

segmenta

descobre microcandidatos

pontua localmente

cria um prompt estruturado

recebe resposta da IA externa

reavalia

decide automaticamente

gera corte + embalagem

registra tudo como dado

O sistema deve ser bom em achar:

momento forte

virada

cena

resposta inesperada

payoff

conflito

revelação

tensão

momento humano

clipe que segura retenção

Ele não pode ficar preso só em:

frase forte

nome famoso

headline

número

polêmica declarada

Modos do sistema
1. Standalone

Tudo local.

Serve para:

fallback

rascunho rápido

operar sem depender de ninguém

2. Híbrido
Esse é o modo principal.

O sistema prepara o melhor material possível.
Você leva para a IA externa.
Ela devolve análise estruturada.
O sistema reavalia e decide.

3. Online

Futuro opcional.
Não é prioridade.

Filosofia de validação

Você corrigiu isso muito bem:

A validação não deve depender de “você gostar ou não”.

O sistema deve tomar a decisão.

Então cada corte passa por:

análise local

análise da IA externa

reanálise local

nota final

decisão automática

Regra de decisão

>= 9.0 → aprovado para corte

7.0 até 8.9 → refazer / melhorar

< 7.0 → descartar

Esse é o centro do projeto.

Módulos principais
1. ingest.py

Responsável por:

registrar projeto

copiar vídeo original

extrair metadados básicos

gerar ID do projeto

2. transcribe.py

Responsável por:

chamar faster-whisper

detectar idioma

gerar transcrição em segmentos

salvar texto e timestamps

3. transcription_quality.py

Responsável por:

detectar qualidade da transcrição

identificar trechos corrompidos

medir confiança por janela

marcar trechos suspeitos

Isso é essencial

Porque transcrição ruim contamina tudo.

4. segment.py

Responsável por:

usar pausas

usar fim de frase

usar limites naturais de fala

criar blocos coerentes

Não em janelas burras apenas.

5. candidate_engine.py

Responsável por:

gerar microcandidatos dentro dos blocos

combinar frase + ritmo + contexto

produzir candidatos de diferentes durações

Faixas que eu usaria

curto: 12–22 s

médio: 22–38 s

longo: 38–60 s

Faixas por modo

headline: 12–28 s

moment: 18–42 s

balanced: 15–35 s

6. scoring_engine.py

Responsável por:

calcular subnotas

produzir score final local

gerar score por plataforma

Subnotas

hook_strength

moment_strength

retention_score

shareability

commentability

clarity_score

transcription_quality

duration_fit

platform_fit

redundancy_penalty

7. platform_engine.py

Responsável por:

dizer qual corte combina mais com:

TikTok

Reels

Shorts

sugerir bucket de duração ideal

sugerir formato editorial

8. hybrid_prompt.py

Responsável por:

pegar tudo o que o sistema já sabe

montar o prompt mais forte possível

enviar contexto estruturado para a IA externa

9. hybrid_parser.py

Responsável por:

validar o JSON da IA externa

extrair campos

rejeitar schema ruim

padronizar o retorno

10. decision_engine.py

Responsável por:

combinar score local com externo

recalcular nota final

decidir:

approved

retry

rejected

11. post_pack.py

Responsável por:

gerar o pacote social do corte

preparar o que será usado no post

12. output_manager.py

Responsável por:

organizar tudo por projeto e por corte

impedir a bagunça das plataformas misturadas

13. learning_engine.py

Responsável por:

receber dados reais de performance

ajustar pesos no futuro

aprender sem depender de modelo pesado logo no início

A nota final

A nota final não deve ser mágica.

Ela vem de subnotas.

Fórmula proposta
final_score =
(local_score * 0.50) +
(external_score * 0.30) +
(platform_fit * 0.10) +
(duration_fit * 0.05) +
(transcription_quality * 0.05)

Isso é calibrável depois.

Plataforma e duração

Você está certo: isso precisa ser central.

Então o sistema deve ter score separado por:

TikTok

Mais tolerante a variações, mas forte em:

scroll stop

impacto

conversa

provocação

Reels

Mais dependente de:

embalagem

limpeza

ritmo

leitura visual

Shorts

Mais dependente de:

payoff

retenção

arco enxuto

E cada corte deve ter:

tiktok_fit

reels_fit

shorts_fit

Não score único genérico.
